# ユーザーニーズ洗い出し

## 目的
[01_ペルソナ定義](./01_ペルソナ定義.md) と [02_ユーザージャーニーマップ](./02_ユーザージャーニーマップ.md) をもとに、ユーザーニーズを洗い出し、LP（Stage1）で満たすべき要件を明確化する。

---

## 1. ニーズの分類（顕在ニーズ / 潜在ニーズ）

### 顕在ニーズ（ユーザー自身が自覚している）
- 自分に合うキャリア・適性を知りたい
- 短時間・手軽に診断したい
- 無料で気軽に試したい
- スマホで完結させたい
- 結果を具体的な行動につなげたい

### 潜在ニーズ（自覚しづらいが満たすと刺さる）
- 漠然とした不安を「整理された安心」に変えたい
- 他人と比較せず、自分基準で納得したい
- 売り込まれずに、中立的なアドバイスが欲しい
- 自分の強みを「言葉」にして人に語れるようにしたい

## 6. 優先度の高いニーズ（Stage1での重点）

ジャーニーのKGI（CTA押下）に直結する順に重点を置く。

1. **手軽さの訴求（F2 / E1）** — 行動の最大の後押し
2. **価値の即時理解（F1 / E2）** — 離脱防止の最重要要素
3. **自分ごと化（F3 / E4）** — 興味喚起・共感
4. **利用イメージの可視化（F4）** — 不安解消
5. **迷わせないCTA（F5）** — 行動の最終トリガー

> **示唆**: Stage1のLPは「手軽さ」と「価値の即時理解」を軸に設計し、各セクションでニーズを段階的に満たしてCTA押下へ導く。
