# AUTH-01 Google認証による会員制機能

## 1. ユーザーニーズ

| ID | ニーズ | 背景 |
|----|--------|------|
| UN-01 | Googleアカウントで手軽にログインしたい | メール/password 登録の手間を避け、既存 Google アカウントで即利用したい |
| UN-02 | ログイン状態を維持したい | 再訪問時に毎回認証せず、セッションでログイン状態を保ちたい |
| UN-03 | 会員専用エリアにアクセスしたい | ダッシュボードで自分向けの情報・導線を確認したい |
| UN-04 | プロフィールを確認・編集したい | 表示名などを自分で更新し、サービス上の自分の情報を管理したい |
| UN-05 | 未ログイン時は保護ページに入れないでほしい | 会員専用ページは認証済みユーザーのみアクセス可能であるべき |

---

## 2. 仕様要件

### 2.1 認証

| 項目 | 要件 |
|------|------|
| 方式 | NextAuth.js v5（Auth.js）+ Google OAuth 2.0 |
| セッション | **JWT 方式**（`session.strategy: "jwt"`） |
| ユーザー永続化 | MongoDB Adapter（users / accounts コレクション） |
| 保護ルート | `/dashboard`, `/profile` — 未ログイン時 `/auth/signin` へリダイレクト |
| ログイン後遷移 | `/dashboard` |

### 2.2 画面

#### `/auth/signin` — ログインページ

- Google ログインボタン（MUI Button）
- ログイン成功後 `/dashboard` へリダイレクト
- LP と統一したダーク宇宙テーマ

#### `/dashboard` — 会員専用トップ

- ユーザー名・アバター表示（MUI Avatar）
- ナビゲーション（ダッシュボード / プロフィール / ログアウト）
- 会員向けウェルカムメッセージ

#### `/profile` — プロフィールページ

- ユーザー情報表示（名前・メール・アバター）
- 名前編集フォーム（MUI TextField）
- 保存ボタンで PUT `/api/user/profile` を呼び出し

### 2.3 API

| メソッド | パス | 説明 | 認証 |
|----------|------|------|------|
| GET/POST | `/api/auth/[...nextauth]` | NextAuth.js 認証エンドポイント | - |
| GET | `/api/user/profile` | ログインユーザーのプロフィール取得 | 必須 |
| PUT | `/api/user/profile` | プロフィール（名前）更新 | 必須 |

### 2.4 データモデル — User

MongoDB `users` コレクション（NextAuth Adapter 標準 + 拡張）

| フィールド | 型 | 説明 |
|------------|-----|------|
| `_id` | ObjectId | 主キー |
| `name` | string | 表示名 |
| `email` | string | メールアドレス |
| `image` | string | Google アバター URL |
| `emailVerified` | Date \| null | メール確認日時 |
| `createdAt` | Date | 初回登録日時（拡張） |
| `updatedAt` | Date | 最終更新日時（拡張） |

### 2.5 技術要件

| 項目 | 採用 |
|------|------|
| 認証 | next-auth v5 |
| DB Adapter | @auth/mongodb-adapter |
| DB クライアント | mongodb |
| UI | MUI（Button, Card, Avatar, TextField） |
| 環境変数 | `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `AUTH_SECRET`, `MONGODB_URI` |

## 3. 処理手順設計

### 3.1 全体アーキテクチャ

```
[Browser]
   │
   ├─ GET /auth/signin ──→ signIn("google") ──→ Google OAuth
   │                                              │
   │                                              ▼
   ├─ /api/auth/[...nextauth] ←─────────── コールバック（JWT 発行）
   │         │
   │         └── MongoDB Adapter → users / accounts 保存
   │
   ├─ middleware ──→ /dashboard, /profile 保護（未ログイン → /auth/signin）
   │
   ├─ GET /api/user/profile ──→ auth() で JWT 検証 → MongoDB から User 取得
   │
   └─ PUT /api/user/profile ──→ auth() で JWT 検証 → name / updatedAt 更新
```

### 3.2 実装手順

| 順序 | 作業 | 成果物 |
|:----:|------|--------|
| 1 | パッケージ追加 | `next-auth`, `@auth/mongodb-adapter`, `mongodb` |
| 2 | MongoDB 接続 | `lib/mongodb.ts` |
| 3 | Auth 設定 | `auth.ts`（Google Provider, JWT, Adapter, callbacks） |
| 4 | 型拡張 | `types/next-auth.d.ts` |
| 5 | API ルート | `app/api/auth/[...nextauth]/route.ts` |
| 6 | プロフィール API | `app/api/user/profile/route.ts` |
| 7 | ミドルウェア | `middleware.ts` |
| 8 | ログインページ | `app/auth/signin/page.tsx` |
| 9 | 会員ページ | `app/dashboard/page.tsx`, `app/profile/page.tsx` |
| 10 | 共通 UI | `MemberNav`, `ProfileForm` |
| 11 | 環境変数テンプレ | `.env.example` |

### 3.3 認証フロー（シーケンス）

```
User → /auth/signin → Google ログインボタン
  → Google 認可画面 → 同意
  → /api/auth/callback/google
  → Adapter: users に upsert（createdAt 設定）
  → JWT 発行 → Cookie 保存
  → /dashboard へリダイレクト
```

### 3.4 プロフィール更新フロー

```
User → /profile → フォームで名前編集 → 保存
  → PUT /api/user/profile { name }
  → auth() で session.user.id 取得
  → MongoDB users.updateOne({ _id }, { name, updatedAt })
  → 200 + 更新後データ返却
  → UI に反映
```

### 3.5 ミドルウェア判定

```
リクエスト pathname
  ├─ /dashboard または /profile で始まる
  │     ├─ auth あり → next()
  │     └─ auth なし → redirect(/auth/signin?callbackUrl=...)
  └─ その他 → next()
```

### 3.6 受け入れ条件（Done の定義）

- [ ] Google ログインで `/dashboard` に遷移できる
- [ ] 未ログインで `/dashboard`, `/profile` にアクセスすると `/auth/signin` へリダイレクト
- [ ] ダッシュボードにユーザー名・アバターが表示される
- [ ] プロフィールページで名前を編集・保存できる
- [ ] GET/PUT `/api/user/profile` が認証なしで 401 を返す
- [ ] `npm run build` が成功する

## 4. 環境変数

```env
# Google OAuth（Google Cloud Console で取得）
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=

# NextAuth.js（openssl rand -base64 32 で生成）
AUTH_SECRET=

# MongoDB Atlas 等
MONGODB_URI=

# 本番 URL（Vercel デプロイ時）
AUTH_URL=https://your-domain.vercel.app
```
