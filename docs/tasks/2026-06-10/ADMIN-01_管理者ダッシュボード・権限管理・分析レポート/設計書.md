# ADMIN-01 管理者ダッシュボード・権限管理・分析レポート

## 1. ユーザーニーズ

| ペルソナ | ニーズ |
|---------|--------|
| サービス運営者（admin） | 診断利用状況・ユーザー数を一目で把握したい |
| サービス運営者（admin） | 問題ユーザーのロール変更・詳細確認がしたい |
| サービス運営者（admin） | 診断トレンド（日別/週別/月別）を分析したい |
| サービス運営者（admin） | システム障害・エラーを早期に検知したい |
| 一般ユーザー（user） | 管理画面にアクセスできない（セキュリティ） |

## 2. 仕様要件

### 2.1 権限管理

| 項目 | 内容 |
|------|------|
| ロール | `admin` / `user`（MongoDB `users.role`、デフォルト `user`） |
| セッション | NextAuth JWT に `role` を載せる |
| 保護 | `/admin/*` は middleware で admin 必須 |
| API | `/api/admin/*` は各 Route で admin チェック |

> **注**: 要件の Mongoose は本プロジェクト方針に合わせ **MongoDB ネイティブドライバー** で実装する。

### 2.2 画面

| パス | 機能 |
|------|------|
| `/admin` | KPI（本日/総診断数、アクティブユーザー）、直近診断5件、システムステータス、クイックリンク |
| `/admin/users` | DataGrid 一覧、検索・ロールフィルタ、ロール変更、詳細ダイアログ |
| `/admin/analytics` | recharts グラフ、TOP10、完了率、属性別傾向、CSV エクスポート |
| `/admin/errors` | エラーログ一覧（MongoDB + Sentry 連携） |

### 2.3 API

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/api/admin/dashboard` | ダッシュボード集計 |
| GET | `/api/admin/users` | ユーザー一覧（page, limit, search, role） |
| GET | `/api/admin/users/[id]` | ユーザー詳細 + 診断履歴 |
| PATCH | `/api/admin/users/[id]` | ロール変更 |
| GET | `/api/admin/analytics` | 分析データ（period: day/week/month） |
| GET | `/api/admin/analytics/export` | CSV ダウンロード |
| GET | `/api/admin/errors` | エラーログ一覧 |

### 2.4 Sentry / 通知

- `@sentry/nextjs` でクライアント・サーバー・Edge を計測
- `beforeSend` で MongoDB `error_logs` にも保存
- オプション: `SLACK_WEBHOOK_URL` / `DISCORD_WEBHOOK_URL` 設定時に通知

### 2.5 環境変数（追加分）

```env
SENTRY_DSN=
SENTRY_AUTH_TOKEN=          # Sentry API 利用時（任意）
SLACK_WEBHOOK_URL=          # 任意
DISCORD_WEBHOOK_URL=        # 任意
ADMIN_BOOTSTRAP_EMAIL=      # 初回 admin 昇格用（任意）
```

## 3. データモデル

### users（拡張）

```typescript
{
  role: "admin" | "user",     // default: "user"
  lastLoginAt?: Date,
  // 既存: name, email, image, gender, ageRange, occupation, ...
}
```

### error_logs（新規）

```typescript
{
  message: string,
  stack?: string,
  url?: string,
  userId?: string,
  level: "error" | "warning",
  sentryEventId?: string,
  createdAt: Date,
}
```

## 4. 処理手順設計

### Phase 1: 基盤

1. `lib/user/types.ts` — `UserRole` 型定義
2. `auth.ts` / `auth.config.ts` — createUser で role 付与、signIn で lastLoginAt、JWT/session に role
3. `middleware.ts` — `/admin` 保護
4. `lib/admin/require-admin.ts` — API 用 admin ガード
5. `lib/admin/server.ts` — 集計・一覧クエリ

### Phase 2: API

1. `/api/admin/dashboard`
2. `/api/admin/users` + `/api/admin/users/[id]`
3. `/api/admin/analytics` + export
4. `/api/admin/errors`

### Phase 3: Sentry

1. `instrumentation.ts`, `sentry.*.config.ts`
2. `next.config.ts` を `withSentryConfig` でラップ
3. `lib/sentry/error-log.ts` — DB 保存 + Webhook 通知

### Phase 4: UI

1. `app/admin/layout.tsx` + `AdminNav`
2. 各ページ + クライアントコンポーネント（DataGrid, recharts）
3. MUI ダークテーマ統一（既存 glassCard スタイル）

### Phase 5: 検証

1. `npm run build` 成功確認
2. admin 以外が `/admin` にアクセス → リダイレクト
3. admin ユーザーで各画面表示

## 5. 初回 admin 設定手順

MongoDB で対象ユーザーの `role` を `"admin"` に更新するか、環境変数 `ADMIN_BOOTSTRAP_EMAIL` を設定して初回ログイン時に自動昇格させる。

```javascript
db.users.updateOne({ email: "your@email.com" }, { $set: { role: "admin" } })
```
