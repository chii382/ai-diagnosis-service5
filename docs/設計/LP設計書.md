## 2. デザインコンセプト（SpaceXオマージュ）

SpaceX公式サイトの世界観を、AIキャリア診断のLPへ落とし込む。

| SpaceXの特徴 | 本LPでの解釈 |
|--------------|--------------|
| 全画面パネルをスクロールで切り替える構成 | 各セクションを `100svh` のフルブリードパネルにし、スクロールスナップで切り替え |
| 漆黒の背景＋宇宙的なビジュアル | 黒背景（#000）＋星空（CSS starfield）＋青系の放射グロー |
| 余白の効いたミニマルなUI | 装飾を削ぎ落とし、テキストとCTAのみで構成 |
| 大文字＋レタースペーシングのタイポ | 英字の「eyebrow（小見出し）」を大文字＋字間広めで配置（例：`AI CAREER DIAGNOSIS`） |
| 細い枠線のテキストボタン | アウトライン／ソリッドの2種。角丸ほぼなし・大文字・字間広め |
| 写真／動画のフルブリード背景 | 各パネルに宇宙的な背景画像（生成画像）をフルブリードで配置。暗色グラデのオーバーレイで可読性を確保し、Ken Burns（ゆっくりズーム）で動画的な動きを付与 |
| 下部に小さなスクロール誘導 | ヒーロー下部に「SCROLL」＋下向きシェブロン（バウンス） |

### トーン&マナー
- **配色**：黒ベースのモノクロ＋青の差し色（グローのみ）。極力モノトーンを保つ。
- **余白**：大胆に取る。1画面1メッセージ。
- **動き**：星のtwinkle、シェブロンのbounce、スクロールスナップ、ホバーの微変化のみ（過剰演出は避ける）。

## 4. 画面全体のワイヤーフレーム

SpaceXのように、縦に連なる全画面パネルをスクロールで見せる。

```
┌─────────────────────────────────────────────┐
│ Header（透明・固定）  ブランド左 / ナビ右       │  ← オーバーレイ
├─────────────────────────────────────────────┤
│ ① HERO  （100svh / 星空＋グロー / 中央）        │
│            AI CAREER DIAGNOSIS                │  ← eyebrow(大文字)
│        5問でわかる、あなたのキャリア             │  ← 特大見出し
│     AIがあなたに最適なキャリアロードマップを…     │  ← サブコピー
│            [ 無料で診断を始める ]               │  ← Solidボタン
│                  ∨ SCROLL                     │  ← スクロール誘導
├─────────────────────────────────────────────┤
│ ② FEATURE 01 （100svh / 青グロー / 左下寄せ）    │
│   FEATURE 01                                  │
│   たった5問・3分で完了                          │
│   忙しくてもスキマ時間で…   [ 詳しく ]           │
├─────────────────────────────────────────────┤
│ ③ FEATURE 02 （100svh / 藍グロー / 左下寄せ）    │
│   AIが深く分析 …                               │
├─────────────────────────────────────────────┤
│ ④ FEATURE 03 （100svh / 青緑グロー / 左下寄せ）  │
│   パーソナライズされた提案 …                     │
├─────────────────────────────────────────────┤
│ ⑤ HOW IT WORKS（3ステップ / 中央 / 番号大）      │
│   01 質問に回答 — 02 AI分析 — 03 結果表示        │
├─────────────────────────────────────────────┤
│ ⑥ FINAL CTA （100svh / 強グロー / 中央）         │
│   さあ、あなたのキャリアを見つけよう             │
│            [ 無料で診断を始める ]               │
├─────────────────────────────────────────────┤
│ ⑦ Footer（黒・極細ボーダー・小さく）             │
└─────────────────────────────────────────────┘
```

### レスポンシブ方針

| ブレークポイント | 想定デバイス | 主な変化 |
|------------------|--------------|----------|
| xs（〜600px） | スマホ | フォント縮小、ヘッダーナビ非表示、ステップ縦積み、左右余白を詰める |
| sm/md（600〜900px） | タブレット | 余白調整、ステップは縦/横を最適化 |
| lg〜（900px〜） | PC | 見出し特大、ステップ横3列、左下寄せの余白を広く |

- 各パネルは `minHeight: 100svh`（モバイルのアドレスバー対策に `svh`）。
- 横スクロールを発生させない（`overflow-x: hidden`）。

## 5. セクション詳細設計

### グローバル（`globals.css` / `theme.ts`）
- `body`：背景 `#000`、文字 `#fff`、`overflow-x: hidden`。
- `html`：`scroll-behavior: smooth`、`scroll-snap-type: y proximity`。
- `.snap`：`scroll-snap-align: start`（全画面パネルに付与）。
- `.stars`：複数の `radial-gradient` を重ねた星空＋`twinkle` アニメ（補助演出）。
- `.bg-zoom`：背景画像を `kenburns`（26s ゆっくり拡大）でアニメーションさせ、動画的な動きを付与。
- `@keyframes twinkle / bounce / kenburns` を定義。`prefers-reduced-motion` で各アニメを停止。

### 背景画像コンポーネント（`common/PanelBackground.tsx`）
- 引数 `src`（画像パス）/ `overlay`（暗色グラデ）/ `position`（表示位置）。
- 構成：①絶対配置の `.bg-zoom`（背景画像・cover・ズーム）→ ②絶対配置のオーバーレイ → ③コンテンツ（`zIndex:1`）。

### 背景画像一覧（`public/images/`）
SpaceX的な「壮大さ＋これから飛び立つ躍動感」を狙い、彩度・明度を上げたシネマティックな宇宙ビジュアルに刷新。オーバーレイも以前より軽くして画の迫力を活かす。
| パネル | 画像 | モチーフ |
|--------|------|----------|
| Hero | `hero-cosmos.png` | 地球の地平線から昇る太陽＋天の川（壮大な夜明け／上昇感） |
| Feature 01 | `feature-blue.png` | 青の星座＋散りばめた星々（地球は使わない） |
| Feature 02 | `feature-indigo.png` | 藍紫の星座＋輝く星々 |
| Feature 03 | `feature-teal.png` | 青緑の星座＋きらめく星々 |
| Steps | `steps-dark.png` | 斜めに走る天の川（カードが映える落ち着いた壮大さ） |
| Final CTA | `cta-cosmos.png` | ロケット打ち上げ（炎を上げて宇宙へ飛び立つ＝最大の躍動感） |

### ① Header（`Header.tsx`）
| 要素 | 仕様 |
|------|------|
| 背景 | 透明（スクロール追従の固定オーバーレイ） |
| 左 | ブランド「AIキャリア診断」＋コンパスアイコン（白） |
| 右（PC） | ナビ `FEATURES` `STEPS`（大文字・字間広め・アンカースクロール）＋小さなアウトラインCTA |
| 右（SP） | ナビは省略し、ブランドのみ（or 小CTA） |

### ② Hero（`HeroSection.tsx`）
| 要素 | 仕様 |
|------|------|
| 高さ | `{ xs: 82svh, md: 86svh }`、内容は中央寄せ（スクロールコスト削減のためフル画面より少し低め） |
| 背景 | 黒＋`.stars`＋上部からの放射グロー |
| 背景 | **動画的な宇宙背景**（`CosmicVideoBackground`）。天の川画像のゆっくりパン＋Canvasでリアルタイム描画する星の流れ・流れ星 |
| eyebrow | `AI CAREER DIAGNOSIS`（大文字・字間広め・半透明白） |
| 見出し | 「5問でわかる、あなたのキャリア」（特大・白） |
| サブコピー | 「AIがあなたに最適なキャリアロードマップを提案します」 |
| CTA | Solidボタン「無料で診断を始める」 |

### ③〜⑤ Features（`FeaturesSection.tsx`）
3つのコンパクトパネル（`minHeight: { xs: 420, md: 500 }`）。テキストは**左寄せ＋縦中央**（`alignItems: center`、`py` 対称）で視覚的に安定させる。間延び防止のためフル画面高ではなくスクロールスナップも外し、連続スクロールで複数パネルを把握できる高さに調整。背景画像は `cover` で短い高さに合わせて表示。

| Panel | eyebrow | 見出し | 説明 | 背景グロー |
|-------|---------|--------|------|-----------|
| 01 | FEATURE 01 | たった5問・3分で完了 | 忙しくてもスキマ時間でOK。5つの質問に直感で答えるだけ。 | 青 |
| 02 | FEATURE 02 | AIが深く分析 | 回答からあなたの強み・適性・価値観を多角的に分析。 | 藍 |
| 03 | FEATURE 03 | パーソナライズされた提案 | あなただけのキャリアロードマップを提案。 | 青緑 |

各パネルはアイコン＋eyebrow＋見出し＋説明文のみのシンプル構成（ボタンは設置しない）。

### ⑥ How it works（`StepsSection.tsx`）
| 要素 | 仕様 |
|------|------|
| eyebrow | `HOW IT WORKS` |
| 見出し | 「診断はかんたん3ステップ」 |
| ステップ | `01 質問に回答` / `02 AI分析` / `03 結果表示`。**ネオン枠のカード3枚並び**（PC横3列／SP縦積み）。各カードは中央寄せで「丸アイコン＋STEP番号＋タイトル＋説明」。色は 01=シアン / 02=ピンク / 03=パープル。中央(02)は枠を強調＋少し浮かせ＋「★ ココが核心」バッジで視線を集める（参照イメージの路線カード風） |

### ⑦ Final CTA（`CTASection.tsx`）
| 要素 | 仕様 |
|------|------|
| 高さ | `100svh`、中央寄せ |
| 背景 | **動画的な打ち上げ背景**（`CosmicVideoBackground` + `rise`）。ロケット打ち上げ画像（`center top`配置でロケット全体を見せる）＋Canvasで星の流れ・流れ星をリアルタイム描画。さらに背景を**ゆっくり上昇させるパン**（`bg-rise` アニメ）で打ち上げの上昇感を演出。オーバーレイは上部のみ濃く（テキスト用）、ロケット部分は明るく保つ縦グラデ。テキストは上部寄せ |
| 見出し | 「さあ、あなたのキャリアを見つけよう」 |
| サブ文 | 「5問に答えるだけ。今すぐ無料で始められます」 |
| CTA | Solidボタン「無料で診断を始める」 |

### ⑧ Footer（`Footer.tsx`）
| 要素 | 仕様 |
|------|------|
| 背景 | 黒＋上部に極細ボーダー |
| 内容 | ブランド名 ／ `© 2026 AIキャリア診断` ／ 小さな大文字リンク（プレースホルダ：PRIVACY / TERMS 等、Stage1は非機能） |

## 7. コンポーネント構成

```
app/
├── layout.tsx              … ルートレイアウト（globals.css読込／ダークテーマ適用）
├── globals.css             … 黒背景・星空・スクロールスナップ・keyframes
├── page.tsx                … LP本体（各パネルを縦に配置）
├── theme.ts                … MUIダークテーマ（色・タイポ・グロー定数）
├── providers/
│   └── ThemeRegistry.tsx   … AppRouterCacheProvider + ThemeProvider
└── components/
    ├── Header.tsx
    ├── HeroSection.tsx
    ├── FeaturesSection.tsx  … 全画面パネル×3
    ├── StepsSection.tsx
    ├── CTASection.tsx
    ├── Footer.tsx
    └── common/
        ├── CTAButton.tsx            … solid / outline（"use client"）
        ├── PanelBackground.tsx      … 背景画像＋オーバーレイ（Features/Steps/CTA）
        └── CosmicVideoBackground.tsx … Hero用。Canvasで星の流れ・流れ星をリアルタイム描画（"use client"）
```

### ヒーローの動画的背景（`common/CosmicVideoBackground.tsx`）
- 実際の動画ファイル（mp4）ではなく、**Canvasでリアルタイムに描画する動画的アニメーション**で宇宙を表現。
- 構成：①天の川画像（`bg-zoom`でゆっくりズーム/パン）→ ②Canvas（視差で流れる星々＋twinkle＋たまに流れ星）→ ③暗色オーバーレイ。
- `requestAnimationFrame`で常時描画。`resize`対応・`devicePixelRatio`対応・`prefers-reduced-motion`時は静止描画。
- 将来、実写の宇宙映像（mp4）に差し替えたい場合は、本コンポーネントを `<video autoPlay muted loop playsInline>` に置き換えるだけで対応可能。

### Server / Client 区分
- セクションは基本 Server Component。
- インタラクションを持つ `CTAButton.tsx` のみ Client Component。

## 8. アクセシビリティ / 品質上の注意

- ダーク背景での**コントラスト確保**（本文は `rgba(255,255,255,0.7)` 以上）。
- `100svh` 採用でモバイルのアドレスバーによる見切れを防ぐ。
- スクロールスナップは `proximity`（強制ではない）でUXを損なわない。
- 星空アニメは控えめにし、`prefers-reduced-motion` 配慮（将来対応可）。
